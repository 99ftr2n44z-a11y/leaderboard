# Leaderboard Service

Сервис лидерборда для управления сезонными рейтингами игроков.

## Архитектура

### Компоненты
1. **PostgreSQL** - основное хранилище данных
2. **Redis** - кэш для быстрого чтения
3. **Go сервис** - бизнес-логика и API

### Выбор архитектуры

#### Реализация с PostgreSQL + Redis

**Преимущества:**
- **Консистентность**: Данные хранятся в PostgreSQL, кэш инвалидируется при обновлении
- **Надежность**: Данные сохраняются в PostgreSQL, Redis используется только для кэширования
- **Производительность**: Redis обеспечивает быстрые чтения, PostgreSQL для сложных запросов
- **Восстановление**: Кэш перестраивается автоматически при промахах
- **Простота**: Минимальное количество компонентов

**Недостатки:**
- Задержка при записи из-за обновления в PostgreSQL
- Возможный рассинхрон между кэшем и БД

### Сравнение альтернатив

| Критерий | PostgreSQL + Redis | Redis Sorted Sets | CockroachDB |
|----------|-------------------|-------------------|-------------|
| Консистентность | Высокая | Средняя | Высокая |
| Надежность | Высокая | Низкая (данные в памяти) | Высокая |
| Производительность чтения | Высокая (кэш) | Очень высокая | Средняя |
| Производительность записи | Средняя | Высокая | Средняя |
| Восстановление | Простое | Сложное (снапшоты) | Простое |
| Сложность реализации | Средняя | Низкая | Высокая |
